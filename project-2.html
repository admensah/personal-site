AI Powered Discord Moderation Bot Case Study

The internet can be a toxic place. Prevent doxxing and swatting on your Discrod communities with our automated flagging and content review system.

See the Code





Project Overview

Our solution is designed to combat online harassment and protect communities by addressing issues of doxing and swatting prevalent on platforms like Discord. Recognizing that anyone can become a victim of these harmful practices, we prioritize a zero-tolerance policy. Our algorithm prefers to err on the side of caution, choosing to flag potential instances of doxing and threats of swatting for review rather than risk allowing such incidents to go unchecked.

In our approach, we have devised a system where users can report abuse directly by interacting with our AI bot. The process is simple and user-friendly - users merely need to direct message the bot with a link to the message containing the alleged abuse. They then categorize the abuse and provide any additional context that could be useful for further examination.

Our technology also features automatic detection capabilities, powered by regex matching that scours all group chat messages for patterns of abusive language. A report is instantly generated upon detection of a match, triggering a prompt review. In severe cases, messages are immediately deleted, providing a safer space for the community.

To enhance our detection capabilities, we utilize the OpenAI API and GPT-3.5 Turbo for classifying incoming messages. This cutting-edge AI tool enables us to classify messages as Doxing, Extortion, Threats, Sexual Harassment, Hate Speech, or Bullying, with an added severity rating ranging from 1 to 4. This detailed classification system allows us to respond effectively to each case of abuse.

We have also implemented the Google Perspective API, a fail-safe mechanism that makes our system more resilient and reliable. This API examines messages for their toxicity score, thus providing another layer of scrutiny for abusive content.

Our moderation flow prioritizes tasks based on the severity of the reported issue. Higher priority reports are dealt with swiftly, and appropriate actions such as warnings, message deletions, or user expulsion are taken.

We understand the importance of data analysis in this context, hence we maintain a comprehensive database. It keeps track of user interactions, records of reports, warnings, and information about instances of false reporting. We've instituted a two-strike rule for false reporting to discourage misuse of the system.

Lastly, we also consider a user's prior conduct when reviewing reports. A message that might not normally be flagged could become a red flag if it's sent by a user with a history of problematic behavior. Through these collective efforts, we strive to provide a safer, more secure, and more respectful online community.

If you intend to use this code for your own discord bot note:

You can follow instructions online to setup the bot, tokens, and the dependencies. Note you must create a tokens.json file in the DiscordBot directory.

You may want to use a Azure, GCP, and AWS instance to run the bot.

Never push your tokens to GitHub or any code sharing site, it will either not allow you or deactivate the token. If you are running the bot in an instance you should ssh into it to place the bot token in the tokens.json file.

There are many more intricacies that only developers will be affordable to navigate in setting up a bot with this complexity. Feel free to contact me for some help!
